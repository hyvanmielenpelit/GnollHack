# Adding Game Content to GnollHack

Adding new monsters, objects, or artifacts requires modifying specific data files following exact macro patterns, then rebuilding auto-generated headers. This guide documents the exact patterns used in the codebase.

## Critical Rules

> **Never hand-edit auto-generated files**: `include/pm.h`, `include/onames.h`, `include/vis_tab.h`, `include/animoff.h`, `include/animtotals.h`. These are regenerated by `makedefs`.

> **Order matters**: Monsters in [monst.c](file:///c:/hmp/GnollHack/src/monst.c) must be grouped by class (S_ANT, S_BAT, etc.) and sorted by ascending strength within each class. Objects in [objects.c](file:///c:/hmp/GnollHack/src/objects.c) must be grouped by object class (WEAPON_CLASS, ARMOR_CLASS, etc.).

> **Rebuild after changes**: After adding monsters or objects, run `makedefs` to regenerate `pm.h` and `onames.h`, then rebuild the native library.

## Adding a Monster

Monsters are defined in [monst.c](file:///c:/hmp/GnollHack/src/monst.c) (6460 lines, ~520KB). Each entry uses one of several macro variants.

### Monster Macro Variants

| Macro | Parameters | Use When |
|-------|-----------|----------|
| `MON(...)` | Base fields + soundset | Standard monster, no custom animation or enlargement |
| `ANIMATED_MON(...)` | + animation, female_anim | Monster with custom tile animation |
| `ENLARGED_MON(...)` | + enlargement, female_enlarge | Monster with oversized tile (2×2, 3×3) |
| `ENLARGED_ANIMATED_MON(...)` | + both | Monster with animation AND enlargement |
| `GENERAL_MON(...)` | + replacement, female_replacement | Full control including replacement info |

### MON() Macro Parameters

From [monst.c:67-68](file:///c:/hmp/GnollHack/src/monst.c#L67):

```c
MON(name, title, description, femalename, commonname,
    symbol,                          // S_ANT, S_BAT, S_DOG, etc.
    LVL(lvl, move, ac, mc, mr, aln), // level, speed, AC, magic cancellation, MR%, alignment
    generation_flags,                // G_GENO | G_SGROUP | frequency(0-7)
    A(attk1, attk2, ..., attk8),     // 8 attack slots (use NO_ATTK for unused)
    SIZ(wt, nut, sound, size, heads, light, material),
    STATS(str, dex, con, int, wis, cha),
    mr1_resistances,                 // MR_FIRE, MR_COLD, MR_POISON, etc.
    mr2_conferred,                   // MR2_* resistances conferred when eaten
    mc_flags,                        // MC_FIRE, MC_POISON, MC_POISONOUS_CORPSE
    m1_flags,                        // M1_FLY, M1_SWIM, M1_ANIMAL, M1_NOHANDS, etc.
    m2_flags,                        // M2_HOSTILE, M2_DOMESTIC, M2_STALK, etc.
    m3_flags,                        // M3_INFRAVISIBLE, M3_INFRAVISION, etc.
    m4_flags, m5_flags, m6_flags, m7_flags, m8_flags,
    difficulty,
    color,                           // CLR_RED, CLR_BROWN, HI_METAL, etc.
    soundset,                        // MONSTER_SOUNDSET_GENERIC or specific
    female_soundset,                 // Usually same as soundset
    soundset_subtype)                // NO_SOUNDSET_SUBTYPE or specific
```

## Adding an Artifact

Artifacts are defined in [artilist.h](file:///c:/hmp/GnollHack/include/artilist.h) (562 lines). Each artifact uses the `A()` macro.

### A() Macro Parameters

```c
A(name, description, hit_description,
  base_object_type, mask_type,
  material, exceptionality, mythic_prefix, mythic_suffix,
  aflags, aflags2,                    // AF_NOGEN, AF_RESTR, AF_INTEL, etc.
  spfx, cspfx,                       // SPFX_* special effects
  mt,                                 // Monster type target (0 = none)
  tohit_dice, tohit_size, tohit_plus, // To-hit bonus
  attack_spec,                        // PHYS(d,s), FIRE(d,s), ELEC(d,s), DRLI(d,s), etc.
  defense, carry, invoke,             // Powers: NO_POWER or specific
  invoke_dur_dice, invoke_dur_size, invoke_dur_plus,
  manacost, repower,
  alignment,                          // A_LAWFUL, A_NEUTRAL, A_CHAOTIC, A_NONE
  role,                               // PM_KNIGHT, PM_VALKYRIE, NON_PM (any)
  race,                               // NON_PM (any)
  cost,                               // Base cost in gold
  glow_color, object_color,           // NO_COLOR or CLR_*
  height, soundset)
```

### Artifact Example

```c

  AF2_NAME_IS_PROPER_NAME,
```

## Auto-Generated Headers

After modifying `monst.c` or `objects.c`, run `makedefs` to regenerate:

| Generated Header | Source | Content |
|-----------------|--------|---------|
| [pm.h](file:///c:/hmp/GnollHack/include/pm.h) | `monst.c` | `#define PM_GIANT_ANT 0`, `#define PM_GIANT_BEETLE 1`, ... |
| [onames.h](file:///c:/hmp/GnollHack/include/onames.h) | `objects.c` | `#define ARROW 0`, `#define ELVEN_ARROW 1`, ... |

**Naming convention**: `PM_` prefix + uppercase name with spaces→underscores for monsters. Object names are uppercased with spaces→underscores.

**Build command**: `makedefs -p` generates `pm.h`, `makedefs -o` generates `onames.h`. See the `build_pipeline` skill for the full build sequence.

## Soundset and Animation Assignment

### Monster Soundsets

Each monster has `soundset` and `female_soundset` fields (last parameters of MON):
- `MONSTER_SOUNDSET_GENERIC` — default sounds
- Specific soundsets defined in [soundset.h](file:///c:/hmp/GnollHack/include/soundset.h) (154KB)
- Soundset data tables in [soundset.c](file:///c:/hmp/GnollHack/src/soundset.c) (1.1MB)

### Object Soundsets

Objects use `OBJECT_SOUNDSET_*` constants:
- `OBJECT_SOUNDSET_NONE`, `OBJECT_SOUNDSET_ARROW`, `OBJECT_SOUNDSET_SHORT_SWORD`, etc.

### Animation Assignment

For animated monsters, use `ANIMATED_MON()` instead of `MON()` and provide animation info:
- Animation definitions in [animation.h](file:///c:/hmp/GnollHack/include/animation.h) (125KB)
- Animation data tables in [animdef.c](file:///c:/hmp/GnollHack/src/animdef.c) (619KB)
- Use `NO_ANIMATION` for no custom animation (the default in `MON()`)

## Step-by-Step: Adding a New Monster

1. **Choose location** in [monst.c](file:///c:/hmp/GnollHack/src/monst.c): Find the correct monster class section (e.g., S_ANT for ants). Insert in ascending strength order.

2. **Write the MON() entry** using the template above. Use `None` for unused string fields. Use `NO_ATTK` for unused attack slots. Use `NO_SOUNDSET_SUBTYPE` for soundset subtype.

3. **Run makedefs**: `makedefs -p` to regenerate `pm.h`. Your new monster gets a `PM_YOUR_MONSTER` constant.

4. **Add tile art**: Add tile graphics to the tile sheet. Run tile utilities to regenerate tile mapping.

5. **Rebuild**: Rebuild the native C library, then rebuild the MAUI project.

## Step-by-Step: Adding a New Object

1. **Choose location** in [objects.c](file:///c:/hmp/GnollHack/src/objects.c): Find the correct object class section. Use the appropriate macro (WEAPON, ARMOR, RING, etc.).

2. **Write the macro entry** following existing examples in the same section.

3. **Run makedefs**: `makedefs -o` to regenerate `onames.h`. Your object gets a named constant (e.g., `MYTHRIL_CHAIN_MAIL`).

4. **Add tile art and soundset** if needed.

5. **Rebuild**: Same as for monsters.
