# GnollHack Build and Compilation Pipeline

The GnollHack build process compiles native C libraries, pre-processes game data assets, and builds the .NET MAUI frontend application.

## Build Pipeline Overview

```
1. makedefs.exe    → Generates headers + data files (11 flags)
2. levcomp.exe     → Compiles 28 .des files → 135 .lev files
3. dgncomp.exe     → Compiles dungeon.pdf → dungeon binary
4. dlb.exe         → Bundles all data → nhdat archive
5. beforegnollhack → Copies sound banks + tilesets
6. gnollhackwin/droid/ios → Compiles native C library per platform
7. aftergnollhack  → Copies native libs + game data to MAUI platform dirs
8. GnollHackM.csproj → .NET MAUI build with all platform assets
```

## Step 1: makedefs

Compiled from [util/makedefs.c](file:///c:/hmp/GnollHack/util/makedefs.c). Post-build runs [aftermakedefs.proj](file:///c:/hmp/GnollHack/win/win32/vs/aftermakedefs.proj) with 11 flags:

| Flag | Output Files | Source |
|------|-------------|--------|
| `-v` | `include/date.h`, `dat/options` | Build metadata |
| `-o` | `include/onames.h` | Object name constants |
| `-p` | `include/pm.h` | Monster index constants |
| `-z` | `include/vis_tab.h`, `src/vis_tab.c` | Vision tables |
| `-d` | `dat/data` | Description database (from `data.base`) |
| `-r` | `dat/rumors` | Fortune cookies (from `rumors.tru` + `rumors.fal`) |
| `-s` | `dat/engrave`, `dat/epitaph`, `dat/bogusmon` | Random text databases |
| `-q` | `dat/quest.dat` | Quest text (from `quest.txt`) |
| `-h` | `dat/oracles` | Oracle text (from `oracles.txt`) |
| `-e` | `dat/dungeon.pdf` | Preprocessed dungeon def (from `dungeon.def`) |
| `-a` | `include/animoff.h`, `include/animtotals.h` | Animation offset tables |

---

## Step 2: levcomp (Level Compiler)

Compiled from [util/lev_comp.y](file:///c:/hmp/GnollHack/util/lev_comp.y) (yacc/lex parser). Post-build runs [afterlevcomp.proj](file:///c:/hmp/GnollHack/win/win32/vs/afterlevcomp.proj).

* **Input**: 28 `.des` files in [dat/](file:///c:/hmp/GnollHack/dat) (e.g., `castle.des`, `mines.des`, `gehennom.des`, `modron.des`)
* **Output**: 135 `.lev` compiled level files
* GnollHack-unique levels: `modron.des` (Modron levels), `quantum.des`, `bovine.des`

---

## Step 3: dgncomp (Dungeon Compiler)

Post-build runs [afterdgncomp.proj](file:///c:/hmp/GnollHack/win/win32/vs/afterdgncomp.proj).

* **Input**: `dat/dungeon.pdf` (generated by makedefs `-e` from `dungeon.def`)
* **Output**: `dat/dungeon` (binary dungeon structure definition)

---

## Step 4: dlb (Data Librarian)

Post-build runs [afterdlb.proj](file:///c:/hmp/GnollHack/win/win32/vs/afterdlb.proj).

* Bundles all compiled data into a single **`nhdat`** archive file
* Contents: `data`, `oracles`, `options`, `quest.dat`, `rumors`, `engrave`, `epitaph`, `bogusmon`, `tribute`, `help`, `hh`, `cmdhelp`, `keyhelp`, `history`, `opthelp`, `wizhelp`, `dungeon`, `license`, `credits`, all `.lev` files
* Command: `dlb.exe cIf dat/dlb.lst bin/nhdat`
* Copies `nhdat` to MAUI Windows game data directory

## Step 5-6: Native Library Compilation

### Game Library Projects

| Project | Platform | Output | Key Defines |
|---------|----------|--------|-------------|
| gnollhackwin | Windows DLL | `gnollhackwin.dll` → MAUI `Platforms/Windows/libs/` | `GNH_MOBILE;GNH_WIN;USE_TILES;DLB;LIB_GRAPHICS;SAFEPROCS` |
| gnollhackdroid | Android .so | `libgnollhackdroid.so` → MAUI `Platforms/Android/libs/{abi}/` | `GNH_MOBILE;GNH_ANDROID;USE_TILES;DLB;LIB_GRAPHICS;SAFEPROCS` |
| gnollhackios | iOS .a | `libgnollhackios.a` → MAUI `Platforms/iOS/libs/` | `GNH_MOBILE;GNH_IOS;USE_TILES;DLB;LIB_GRAPHICS;SAFEPROCS` |

All define: `NOTTYGRAPHICS;GNOLLHACK_MAIN_PROGRAM;EXIT_THREAD_ON_EXIT;DUMPLOG;DUMPHTML`

### Post-build Steps

| Script | Action |
|--------|--------|
| [aftergnollhack.proj](file:///c:/hmp/GnollHack/win/win32/vs/aftergnollhack.proj) | Copies game files + FMOD DLLs to `bin/` for standalone desktop |
| [aftergnollhackdll.proj](file:///c:/hmp/GnollHack/win/win32/vs/aftergnollhackdll.proj) | Copies game data + FMOD DLLs to MAUI `Platforms/Windows/` |
| [aftergnollhackdroid.proj](file:///c:/hmp/GnollHack/win/win32/vs/aftergnollhackdroid.proj) | Copies `.so` to MAUI `Platforms/Android/libs/{abi}/` |
| [aftergnollhackios.proj](file:///c:/hmp/GnollHack/win/win32/vs/aftergnollhackios.proj) | Copies `.a` from Mac build output to MAUI iOS native refs |
| [beforegnollhack.proj](file:///c:/hmp/GnollHack/win/win32/vs/beforegnollhack.proj) | Pre-build: copies sound banks from `bank/` and tilesets from `tileset/` |

## Step 7: MAUI Application Build

Project: [GnollHackM.csproj](file:///c:/hmp/GnollHack/win/win32/xpl/GnollHackM/GnollHackM.csproj)

### Native Library References per Platform

| Platform | Reference Type | Libraries |
|----------|---------------|------------|
| Android | `<AndroidNativeLibrary>` | `libgnollhackdroid.so`, `libfmod.so`, `libfmodstudio.so`, `libc++_shared.so` |
| iOS | `<NativeReference Kind=Static>` | `libgnollhackios.a`, `libfmod_iphoneos.a`, `libfmodstudio_iphoneos.a` |
| Windows | `<MauiAsset CopyToOutputDirectory=Always>` | `gnollhackwin.dll`, `fmod.dll`, `fmodstudio.dll`, `fsbank.dll` |

### Game Data per Platform

| Platform | Asset Type | Source Directory |
|----------|-----------|------------------|
| Android | `<AndroidAsset>` | `Platforms/Android/gnh/` (nhdat, license, symbols, sysconf, defaults.gnh) |
| iOS | `<BundleResource>` | `Platforms/iOS/gnh/` |
| Windows | `<MauiAsset>` | `Platforms/Windows/gnh/` |

### Build Commands

```powershell
# Windows
dotnet build win/win32/xpl/GnollHackM/GnollHackM.csproj -c Debug -f net10.0-windows10.0.19041.0

# Android
dotnet build win/win32/xpl/GnollHackM/GnollHackM.csproj -c Debug -f net10.0-android

# iOS
dotnet build win/win32/xpl/GnollHackM/GnollHackM.csproj -c Debug -f net10.0-ios
```

## Output Directory Structure

Build output is configured in [dirs.props](file:///c:/hmp/GnollHack/win/win32/vs/dirs.props):

| Variable | Path | Content |
|----------|------|----------|
| `BinDir` | `bin/$(Configuration)/$(Platform)/` | Standalone desktop build output |
| `ToolsDir` | `tools/$(Configuration)/$(Platform)/` | Compiled utilities (makedefs, levcomp, etc.) |
| `MauiWindowsLibsDir` | `...GnollHackM/Platforms/Windows/libs/` | Windows native libs + FMOD |
| `MauiAndroidLibsDir` | `...GnollHackM/Platforms/Android/libs/` | Android .so per ABI |
| `MauiWindowsGnhDir` | `...GnollHackM/Platforms/Windows/gnh/` | Windows game data (nhdat, etc.) |
