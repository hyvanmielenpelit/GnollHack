# Game Data Files

GnollHack uses text-based source data files in [dat/](file:///c:/hmp/GnollHack/dat) that are compiled into binary formats by build-time utilities. Understanding this pipeline is essential for adding levels, modifying game text, or changing dungeon structure.

## Data Flow

```
Source Text Files  →  Build Utilities  →  Compiled Binaries  →  nhdat Archive
     (dat/)           (makedefs,          (dat/*.lev,            (single file
                       levcomp,            dat/dungeon,           bundled by
                       dgncomp)            dat/data, etc.)        dlb)
```

## Text Data Sources

These text files are compiled by `makedefs` into binary lookup databases:

| Source File | makedefs Flag | Output | Description |
|-------------|--------------|--------|-------------|
| `dat/data.base` (273KB) | `-d` | `dat/data` | Object/monster description database |
| `dat/quest.txt` | `-q` | `dat/quest.dat` | Quest dialogue and narrative text |
| `dat/oracles.txt` | `-h` | `dat/oracles` | Oracle consultation messages |
| `dat/rumors.tru` + `dat/rumors.fal` | `-r` | `dat/rumors` | Fortune cookie messages (true + false) |
| `dat/epitaph.txt` | `-s` | `dat/epitaph` | Random gravestone inscriptions |
| `dat/engrave.txt` | `-s` | `dat/engrave` | Random floor engravings |
| `dat/bogusmon.txt` | `-s` | `dat/bogusmon` | Hallucinatory monster names |

## Generated Header Files

These headers are auto-generated by `makedefs` and must NOT be hand-edited:

| File | Flag | Content |
|------|------|----------|
| `include/date.h` | `-v` | Build timestamp and version info |
| `include/onames.h` | `-o` | Object name `#define` constants (e.g., `SCR_ENCHANT_WEAPON`) |
| `include/pm.h` | `-p` | Monster index `#define` constants (e.g., `PM_DRAGON`) |
| `include/vis_tab.h` | `-z` | Vision calculation lookup tables |
| `include/animoff.h` | `-a` | Animation offset tables |
| `include/animtotals.h` | `-a` | Animation total counters |

## The nhdat Archive

All compiled data is bundled into a single `nhdat` file by the `dlb` (Data Librarian) utility:

* **Input list**: Written to `dat/dlb.lst` from the `DlbList` item group in [files.props](file:///c:/hmp/GnollHack/win/win32/vs/files.props)
* **Command**: `dlb.exe cIf dat/dlb.lst bin/nhdat`
* **Output**: Single `nhdat` file distributed with the application
* At runtime, the game reads individual files from within `nhdat` using the `dlb` virtual filesystem API ([include/dlb.h](file:///c:/hmp/GnollHack/include/dlb.h))

## Adding a New Special Level

1. Create or edit a `.des` file in `dat/`
2. If new file: add it to `Desfiles` item group in [files.props](file:///c:/hmp/GnollHack/win/win32/vs/files.props)
3. Add expected `.lev` output(s) to `Levfiles` item group in same file
4. If the level needs a branch entry: edit `dat/dungeon.def`
5. Rebuild: makedefs → levcomp → dgncomp → dlb
6. The `.lev` files will be automatically included in `nhdat`

## Modifying Game Text

1. Edit the source text file (e.g., `dat/rumors.tru` for true rumors)
2. Rebuild with `makedefs` using the appropriate flag (e.g., `-r`)
3. Rebuild `dlb` to update `nhdat`
